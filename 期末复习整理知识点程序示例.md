## 1.1 支线预备知识

### 1.1.1 内联函数 `inline`

```cpp
inline int Square(int x)
{
    return x * x;
}

int main()
{
    cout << Square(5) << endl;   // 25
}
```

看点：`inline` 写在函数定义前，适合短小函数。类内直接定义的成员函数通常也可视作内联。

### 1.1.2 引用

```cpp
void Swap(int& a, int& b)
{
    int t = a;
    a = b;
    b = t;
}

int main()
{
    int x = 3, y = 5;
    Swap(x, y);
    cout << x << " " << y << endl;   // 5 3
}
```

看点：引用是变量别名，形参写 `int&` 后，函数内修改会影响实参。

### 1.1.3 函数重载

```cpp
int Add(int a, int b)
{
    return a + b;
}

double Add(double a, double b)
{
    return a + b;
}

int main()
{
    cout << Add(1, 2) << endl;       // 调 int 版本
    cout << Add(1.2, 3.4) << endl;   // 调 double 版本
}
```

看点：函数名相同，参数个数或类型不同即可重载；只改返回值类型不能构成重载。

## 1.2 类和对象

### 1.2.1 面向对象程序设计的含义

```cpp
class Student {
public:
    void SetScore(int s) { score = s; }
    void Print() { cout << score << endl; }
private:
    int score;
};

int main()
{
    Student stu;       // 对象
    stu.SetScore(90);  // 消息：调用对象的方法
    stu.Print();
}
```

看点：程序围绕对象组织，数据 `score` 和行为 `SetScore/Print` 绑定在类中。

### 1.2.2 对象

```cpp
class Point {
public:
    int x, y;
};

int main()
{
    Point p;
    p.x = 1;
    p.y = 2;
    cout << sizeof(p) << endl;
}
```

看点：类不占对象空间，对象才占内存；`sizeof(p)` 是对象所占空间。

### 1.2.3 类

```cpp
class Circle {
public:
    void Set(double r) { radius = r; }
    double Area() { return 3.14 * radius * radius; }
private:
    double radius;
};

int main()
{
    Circle c;
    c.Set(2);
    cout << c.Area() << endl;
}
```

看点：`class` 描述同类对象的共同属性和行为；默认访问权限是 `private`。

### 1.2.4 封装性

```cpp
class BankAccount {
public:
    void Deposit(int x)
    {
        if (x > 0)
            balance += x;
    }

    int GetBalance() const
    {
        return balance;
    }

private:
    int balance = 0;
};
```

看点：外部不能直接乱改 `balance`，只能通过公有接口访问，这是封装。

### 1.2.5 成员函数

```cpp
class A {
public:
    void Set(int n);       // 类内声明
    int Get() { return x; } // 类内定义
private:
    int x;
};

void A::Set(int n)         // 类外定义要写 A::
{
    x = n;
}

int main()
{
    A a;
    a.Set(10);
    cout << a.Get() << endl;
}
```

看点：类外定义成员函数时，函数名前要加 `类名::`。

### 1.2.6 对象相关指针

```cpp
class Box {
public:
    int value;
    void Print() { cout << value << endl; }
};

int main()
{
    Box b;
    b.value = 7;

    Box* pb = &b;                 // 指向对象的指针
    cout << pb->value << endl;

    int Box::*pm = &Box::value;   // 指向数据成员的指针
    cout << b.*pm << endl;

    void (Box::*pf)() = &Box::Print; // 指向成员函数的指针
    (b.*pf)();
}
```

看点：成员函数指针调用时必须绑定对象：`(对象.*指针)(实参)`。

### 1.2.7 引用对象的三种方法

```cpp
class A {
public:
    int x;
};

int main()
{
    A a;
    A* p = &a;
    A& r = a;

    a.x = 1;   // 对象名 .
    p->x = 2;  // 对象指针 ->
    r.x = 3;   // 对象引用 .

    cout << a.x << endl; // 3
}
```

看点：对象、对象指针、对象引用访问成员的符号不同。

### 1.2.8 类的组合

```cpp
class Point {
public:
    Point(int x0 = 0, int y0 = 0) : x(x0), y(y0) {}
    void Print() { cout << "(" << x << "," << y << ")"; }
private:
    int x, y;
};

class Circle {
public:
    Circle(int x, int y, int r) : center(x, y), radius(r) {}
    void Print()
    {
        center.Print();
        cout << " r=" << radius << endl;
    }
private:
    Point center;  // 内嵌对象
    int radius;
};
```

看点：一个类把另一个类的对象作为成员，就是组合。

## 1.3 继承

### 1.3.1 基本概念

```cpp
class Animal {
public:
    void Eat() { cout << "eat" << endl; }
};

class Dog : public Animal {
public:
    void Bark() { cout << "bark" << endl; }
};

int main()
{
    Dog d;
    d.Eat();   // 继承自 Animal
    d.Bark();  // Dog 新增
}
```

看点：派生类获得基类成员，并可以增加新成员。

### 1.3.2 派生类

```cpp
class Base {
public:
    void Show() { cout << "Base" << endl; }
};

class Derived : public Base {
public:
    void Show() { cout << "Derived" << endl; } // 同名覆盖
    void NewFun() { cout << "new" << endl; }
};
```

看点：派生类可以继承、增加成员，也可以同名覆盖基类成员。

### 1.3.3 二义性问题

多继承二义性明确不考，不重点练。单继承里的同名覆盖仍要会：

```cpp
class Base {
public:
    void Print() { cout << "Base" << endl; }
};

class Derived : public Base {
public:
    void Print() { cout << "Derived" << endl; }
    void Test()
    {
        Print();       // Derived::Print
        Base::Print(); // 指定调用基类版本
    }
};
```

看点：派生类和基类函数同名时，不加限定优先调用派生类版本。

### 1.3.4 虚基类

虚基类通常服务于多继承重复继承问题，明确不考。识别即可：

```cpp
class A {};
class B : virtual public A {};
```

看点：看到 `virtual public A` 这类写法，知道它是虚基类继承；不投入刷题时间。

### 1.3.5 基类与派生类兼容规则

```cpp
class Base {
public:
    void Print() { cout << "Base part" << endl; }
};

class Derived : public Base {
public:
    void Extra() { cout << "Derived part" << endl; }
};

int main()
{
    Derived d;
    Base b = d;     // 派生类对象赋给基类对象，只保留基类部分
    Base& r = d;    // 基类引用绑定派生类对象
    Base* p = &d;   // 基类指针指向派生类对象

    b.Print();
    r.Print();
    p->Print();
}
```

看点：兼容是单向的，公有派生类可以当作基类用，反过来不行。

### 1.3.6 虚函数

```cpp
class Base {
public:
    virtual void Print() { cout << "Base" << endl; }
};

class Derived : public Base {
public:
    void Print() { cout << "Derived" << endl; }
};

int main()
{
    Derived d;
    Base* p = &d;
    p->Print(); // Derived，运行时动态绑定
}
```

看点：基类指针/引用调用虚函数时，按实际对象类型决定调用哪个函数。

### 1.3.7 关联/绑定

```cpp
class Base {
public:
    virtual void f() { cout << "Base::f" << endl; }
};

class Derived : public Base {
public:
    void f() { cout << "Derived::f" << endl; }
};

int main()
{
    Derived d;
    Base b = d;
    Base* p = &d;

    b.f();  // Base::f，按对象名调用，静态关联
    p->f(); // Derived::f，基类指针调用虚函数，动态关联
}
```

看点：动态关联的典型条件是“基类指针/引用 + 虚函数”。

### 1.3.8 虚函数与重载函数对比

```cpp
void Print(int x) { cout << "int " << x << endl; }
void Print(double x) { cout << "double " << x << endl; }

class Base {
public:
    virtual void Show() { cout << "Base" << endl; }
};

class Derived : public Base {
public:
    void Show() { cout << "Derived" << endl; }
};
```

看点：函数重载看参数表，编译时决定；虚函数看实际对象，运行时决定。

### 1.3.9 纯虚函数

```cpp
class Shape {
public:
    virtual double Area() = 0; // 纯虚函数
};

class Rectangle : public Shape {
public:
    Rectangle(double w, double h) : width(w), height(h) {}
    double Area() { return width * height; }
private:
    double width, height;
};
```

看点：`= 0` 表示纯虚函数，基类只规定接口，具体功能由派生类实现。

### 1.3.10 抽象类

```cpp
int main()
{
    // Shape s;          // 错：抽象类不能建立对象
    Rectangle r(3, 4);   // 对所有纯虚函数作出实现后，可以建立对象
    Shape* p = &r;
    cout << p->Area() << endl; // 12
}
```

看点：含纯虚函数的类是抽象类，通常用于统一接口。

## 1.4 静态成员

### 1.4.1 变量存储类型

```cpp
void Count()
{
    int a = 0;        // 每次调用重新初始化
    static int b = 0; // 只初始化一次，生命期到程序结束
    a++;
    b++;
    cout << a << " " << b << endl;
}

int main()
{
    Count(); // 1 1
    Count(); // 1 2
}
```

看点：局部静态变量保留上次调用后的值。

### 1.4.2 类的成员属性

```cpp
class Student {
public:
    Student(int s) : score(s) { count++; }
    void Print() { cout << score << " " << count << endl; }
private:
    int score;           // 对象属性：每个对象一份
    static int count;    // 类属性：所有对象共享
};

int Student::count = 0;
```

看点：普通数据成员属于对象，静态数据成员属于类。

### 1.4.3 静态数据成员

```cpp
class Counter {
public:
    Counter() { num++; }
    static int GetNum() { return num; }
private:
    static int num;
};

int Counter::num = 0; // 类外初始化

int main()
{
    Counter a, b;
    cout << Counter::GetNum() << endl; // 2
}
```

看点：静态数据成员必须在类外定义并初始化。

### 1.4.4 静态成员函数

```cpp
class A {
public:
    static void Print()
    {
        cout << n << endl;
        // cout << x; // 错：静态成员函数不能直接访问非静态成员
    }
private:
    static int n;
    int x;
};

int A::n = 10;
```

看点：静态成员函数没有 `this` 指针，只能直接访问静态成员。

## 1.5 访问属性

### 1.5.1 访问属性关键字

```cpp
class A {
public:
    int pub;
protected:
    int pro;
private:
    int pri;
};
```

看点：`public` 类外可访问，`protected` 派生类可访问，`private` 只有本类可访问。

### 1.5.2 类成员的访问属性

```cpp
class A {
public:
    void Set(int n) { pri = n; }
    int Get() { return pri; }
private:
    int pri;
};

int main()
{
    A a;
    a.Set(5);
    cout << a.Get() << endl;
    // a.pri = 6; // 错：private 类外不可访问
}
```

看点：私有成员常通过公有成员函数间接访问。

### 1.5.3 派生类可访问的成员

```cpp
class Base {
public:
    int pub;
protected:
    int pro;
private:
    int pri;
};

class Derived : public Base {
public:
    void Test()
    {
        pub = 1; // 可以
        pro = 2; // 可以
        // pri = 3; // 错：基类 private 不可直接访问
    }
};
```

看点：派生类成员函数能访问基类的 `public` 和 `protected`，不能直接访问 `private`。

### 1.5.4 派生类对基类访问属性调整

```cpp
class Base {
public:
    void f() {}
protected:
    void g() {}
};

class PublicD : public Base {};
class ProtectedD : protected Base {};
class PrivateD : private Base {};

int main()
{
    PublicD a;
    a.f();       // 可以，public 继承后仍为 public

    ProtectedD b;
    // b.f();    // 错，protected 继承后变 protected

    PrivateD c;
    // c.f();    // 错，private 继承后变 private
}
```

看点：继承方式会改变基类 `public/protected` 成员在派生类中的访问属性。

### 1.5.5 多级派生对访问权限的调整

```cpp
class A {
protected:
    int x;
};

class B : public A {
public:
    void Set(int n) { x = n; }
};

class C : public B {
public:
    void Add() { x++; } // 多级公有继承后，protected 仍可在派生类内访问
};
```

看点：每一级继承都按访问规则调整；基类 `private` 永远不能被派生类直接访问。

### 1.5.6 `using` 用于类成员继承

```cpp
class Base {
public:
    void f() { cout << "Base::f" << endl; }
};

class Derived : private Base {
public:
    using Base::f; // 把 Base::f 重新作为 public 成员暴露
};

int main()
{
    Derived d;
    d.f();
}
```

看点：`using Base::f;` 可以恢复或调整继承成员的访问方式，也可用于引入基类重载函数。

### 1.5.7 友元

```cpp
class Point {
    friend void Print(Point& p);
private:
    int x = 1;
};

void Print(Point& p)
{
    cout << p.x << endl; // 友元函数可访问 private
}

int main()
{
    Point p;
    Print(p);
}
```

看点：友元声明不是成员函数声明；友元关系不继承、不传递、不相互。

## 1.5 构造函数

### 1.5.1 构造函数特点

```cpp
class Point {
public:
    Point()
    {
        x = 0;
        y = 0;
        cout << "construct" << endl;
    }
private:
    int x, y;
};

int main()
{
    Point p; // 建立对象时自动调用构造函数
}
```

看点：构造函数无返回值，函数名和类名相同，建立对象时自动调用。

### 1.5.2 构造函数类型

```cpp
class Point {
public:
    Point() : x(0), y(0) {}                 // 默认构造函数
    Point(int a, int b) : x(a), y(b) {}     // 带参数构造函数
    Point(const Point& p) : x(p.x), y(p.y)  // 拷贝构造函数
    {
        cout << "copy" << endl;
    }
private:
    int x, y;
};

int main()
{
    Point p1;
    Point p2(1, 2);
    Point p3 = p2; // 调拷贝构造
}
```

看点：构造函数可以重载；对象创建时按实参匹配其中一个。

### 1.5.3 带参数构造函数

```cpp
class Point {
public:
    Point(int a = 0, int b = 0) : x(a), y(b) {}
    void Print() { cout << x << "," << y << endl; }
private:
    int x, y;
};

int main()
{
    Point p1;
    Point p2(3);
    Point p3(3, 4);
}
```

看点：默认参数要从右向左给；成员初始化列表写在构造函数小括号后、大括号前。

### 1.5.4 拷贝构造函数

```cpp
class A {
public:
    A(int n = 0) : x(n) {}
    A(const A& a) : x(a.x)
    {
        cout << "copy" << endl;
    }
private:
    int x;
};

void f(A a) {}  // 值传参会调用拷贝构造

int main()
{
    A a1(1);
    A a2 = a1;  // 用已有对象初始化新对象
    f(a1);      // 对象以值传入函数
}
```

看点：`A a2 = a1;` 是拷贝构造；若对象已经存在，再写 `a2 = a1;` 是赋值。

### 1.5.5 继承和组合中的构造函数

```cpp
class Base {
public:
    Base(int n) { cout << "Base " << n << endl; }
};

class Member {
public:
    Member(int n) { cout << "Member " << n << endl; }
};

class Derived : public Base {
public:
    Derived(int a, int b, int c) : Base(a), m(b), x(c)
    {
        cout << "Derived" << endl;
    }
private:
    Member m;
    int x;
};

int main()
{
    Derived d(1, 2, 3);
}
```

看点：派生类构造时先构造基类，再构造成员对象，最后执行派生类构造函数体。

## 1.6 析构函数

### 1.6.1 析构函数概念

```cpp
class A {
public:
    A() { cout << "construct" << endl; }
    ~A() { cout << "destruct" << endl; }
};

int main()
{
    A a;
} // 离开作用域时自动调用析构函数
```

看点：析构函数常用于释放资源，名字是 `~类名()`。

### 1.6.2 析构函数特点

```cpp
class Buffer {
public:
    Buffer() { p = new int[10]; }
    ~Buffer() { delete [] p; }
private:
    int* p;
};
```

看点：析构函数没有返回值、没有参数、不能重载。`new []` 对应 `delete []`。

### 1.6.3 虚析构函数

```cpp
class Base {
public:
    virtual ~Base() { cout << "~Base" << endl; }
};

class Derived : public Base {
public:
    ~Derived() { cout << "~Derived" << endl; }
};

int main()
{
    Base* p = new Derived;
    delete p; // 先 ~Derived，再 ~Base
}
```

看点：通过基类指针删除派生类对象时，基类析构函数应声明为 `virtual`。

## 1.7 构造函数和析构函数的执行顺序

### 1.7.1 一般原则

```cpp
class A {
public:
    A(char c) : name(c) { cout << "A" << name << " "; }
    ~A() { cout << "~A" << name << " "; }
private:
    char name;
};

int main()
{
    A a('1');
    A b('2');
} // 输出构造 A1 A2；析构顺序相反 ~A2 ~A1
```

看点：先构造的后析构，后构造的先析构。

### 1.7.2 函数参数传递中的顺序

```cpp
class A {
public:
    A(char c) : name(c) { cout << "C" << name << " "; }
    A(const A& a) : name(a.name) { cout << "Copy" << name << " "; }
    ~A() { cout << "D" << name << " "; }
private:
    char name;
};

void f(A x, A y) {}

int main()
{
    A a('a'), b('b');
    f(a, b);
}
```

看点：资料按 Windows/VS2010 环境总结，函数实参常从右到左构造；考试按课件口径判断。现代编译器可能有差异，不拿它做跨环境验证。

### 1.7.3 继承和嵌套时的顺序

```cpp
class Base {
public:
    Base() { cout << "Base "; }
    ~Base() { cout << "~Base "; }
};

class Member {
public:
    Member() { cout << "Member "; }
    ~Member() { cout << "~Member "; }
};

class Derived : public Base {
public:
    Derived() { cout << "Derived "; }
    ~Derived() { cout << "~Derived "; }
private:
    Member m;
};

int main()
{
    Derived d;
}
```

看点：构造顺序：基类、成员对象、派生类；析构顺序相反。

## 1.8 对象的空间分配

### 1.8.1 对象的构建

```cpp
class A {
public:
    A(int n = 0) { cout << n << endl; }
};

int main()
{
    A a;      // 默认构造
    A b(10);  // 带参构造
}
```

看点：定义对象时分配空间并调用构造函数。

### 1.8.2 对象数组静态空间分配

```cpp
class A {
public:
    A(int n = 0) { cout << "A" << n << " "; }
};

int main()
{
```

### 1.8.6 对象指针成员的拷贝

```cpp
#include <cstring>
#include <iostream>
using namespace std;

class String {
public:
    String(const char* s = "")
    {
        p = new char[strlen(s) + 1];
        strcpy(p, s);
    }

    String(const String& s)
    {
        p = new char[strlen(s.p) + 1];
        strcpy(p, s.p);
    }

    String& operator=(const String& s)
    {
        if (this != &s) {
            delete [] p;
            p = new char[strlen(s.p) + 1];
            strcpy(p, s.p);
        }
        return *this;
    }

    ~String()
    {
        delete [] p;
    }

private:
    char* p;
};
```

看点：类中有指针成员并管理动态空间时，要写深拷贝：拷贝构造、赋值运算符、析构函数。

### 3.2.2 运算符重载

```cpp
class Complex {
public:
```

看点：`operator+` 是函数名；表达式 `a + b` 会转为调用 `a.operator+(b)`。

### 3.2.3 重载的限制

```cpp

// operator.
// operator::
// operator?:
// operator sizeof
```

看点：运算符重载不能改变操作数个数、优先级、结合性，也不能给重载运算符设置默认参数。

### 3.2.4 运算符重载方式及规则

成员函数版本：

```cpp
class Complex {
public:
    Complex(double r = 0, double i = 0) : real(r), imag(i) {}
    Complex operator+(const Complex& c)
    {
        return Complex(real + c.real, imag + c.imag);
    }
private:
    double real, imag;
};
```

友元函数版本：

```cpp
class Complex {
public:
    Complex(double r = 0, double i = 0) : real(r), imag(i) {}
    friend Complex operator+(const Complex& a, const Complex& b);
private:
    double real, imag;
};

Complex operator+(const Complex& a, const Complex& b)
{
    return Complex(a.real + b.real, a.imag + b.imag);
}
```

看点：成员函数左操作数是 `this`；友元函数通过形参决定两个操作数。

### 3.2.5 特殊运算符重载

前缀/后缀 `++`：

```cpp
class Counter {
public:
    Counter(int n = 0) : value(n) {}

    Counter& operator++()      // 前缀 ++c
    {
        value++;
        return *this;
    }

    Counter operator++(int)    // 后缀 c++
    {
        Counter old = *this;
        value++;
        return old;
    }

private:
    int value;
};
```

流运算符：

```cpp
class Point {
public:
    Point(int a = 0, int b = 0) : x(a), y(b) {}
    friend istream& operator>>(istream& in, Point& p);
    friend ostream& operator<<(ostream& out, const Point& p);
private:
    int x, y;
};

istream& operator>>(istream& in, Point& p)
{
    in >> p.x >> p.y;
    return in;
}

ostream& operator<<(ostream& out, const Point& p)
{
    out << "(" << p.x << "," << p.y << ")";
    return out;
}
```

下标和函数调用：

```cpp
class Array {
public:
    int& operator[](int i) { return a[i]; }
    int operator()(int i, int j) { return a[i] + a[j]; }
private:
    int a[10];
};
```

看点：后缀 `++` 多一个哑元 `int`；`<<`、`>>` 返回流引用；`[]`、`()`, `=`, `->` 只能重载为成员函数。

### 3.2.6 不同类型数据间转换

转换构造函数：

```cpp

    double real, imag;

```

类型转换函数：

看点：转换构造函数是“别的类型 -> 本类”；类型转换函数是“本类 -> 别的类型”。

### 3.2.7 赋值运算符 `=`

```cpp
class A {
public:
    A(int n = 0) : x(n) {}
    A(const A& a) : x(a.x) { cout << "copy" << endl; }

    A& operator=(const A& a)
    {
        if (this != &a)
            x = a.x;
        return *this;
    }

private:
    int x;
};

int main()
{

    A b = a; // 拷贝构造：新建 b

}
```

看点：`类名 新对象 = 原对象;` 是拷贝构造；`已有对象 = 原对象;` 是赋值。

## 3.3 数据保护的常类型

### 3.3.1 一般原则

```cpp
int main()
{
    int a = 1, b = 2;

    const int* p1 = &a; // 指向常量的指针，不能通过 p1 改 a
    p1 = &b;            // 但 p1 可以改指向

    int* const p2 = &a; // 指针常量，不能改指向
    *p2 = 3;            // 但可以通过 p2 改 a
}
```

看点：`const` 优先修饰左边最近的符号；最左边没有可修饰对象时修饰右边。

### 3.3.2 基本数据类型的三种常变量

```cpp
int main()
{
    const int a = 10;      // 常变量
    int x = 1, y = 2;

    const int* p = &x;     // 常量指针：不能通过 p 改值
    p = &y;                // 可以改指向

    int* const q = &x;     // 指针常量：不能改指向
    *q = 5;                // 可以改所指内容

    const int* const r = &x; // 指向常量的常指针
}
```

看点：区分“所指内容不能改”和“指针指向不能改”。

### 3.3.3 类与对象的六种常类型

```cpp
class Point {
public:
    Point(int a, int b) : x(a), y(b), id(0) {}

    void Move(int dx, int dy)
    {
        x += dx;
        y += dy;
    }

    void Print() const
    {
        cout << x << "," << y << endl;
        // x++; // 错：常成员函数不能修改普通成员
    }

private:
    int x, y;
    const int id; // 常成员变量，只能在初始化列表中初始化
};

int main()
{
    const Point p1(1, 2);  // 常对象
    p1.Print();
    // p1.Move(1, 1);      // 错：常对象不能调用非常成员函数

    Point p2(3, 4);
    const Point* p = &p2;  // 常对象指针
    Point* const q = &p2;  // 对象指针常量
    const Point& r = p2;   // 对象常引用
}
```

看点：常对象只能调用常成员函数；常成员变量必须用初始化列表初始化。

### 3.3.4 共用数据保护策略

```cpp
class Point {
public:
    Point(int a = 0, int b = 0) : x(a), y(b) {}
    int GetX() const { return x; }
private:
    int x, y;
};

void PrintPoint(const Point& p)
{
    cout << p.GetX() << endl;
}

int main()
{
    Point p(1, 2);
    PrintPoint(p);
}
```

看点：不希望函数修改对象时，用 `const 类名&` 传参；只读成员函数加 `const`。
