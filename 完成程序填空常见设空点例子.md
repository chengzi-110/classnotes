# 完成程序填空常见设空点例子

来源说明：优先参考 `往年题与练习题/2022A 卷.pdf`、`往年题与练习题/2022答案.pdf` 和项目内专题练习题；没有合适原题时补充自拟短例。已按不考范围过滤：多继承、模板、异常处理、STL 不纳入重点。

## 0.3.1 函数声明/定义中挖出一行的一部分

### 1. 虚函数 `= 0` 标记

空常出现在虚函数声明末尾。

```cpp
class Shape {
public:
    virtual double Area() ________;
};
```

应填：

```cpp
= 0
```

完整形式：

```cpp
virtual double Area() = 0;
```

判断依据：含纯虚函数的类是抽象类，题目若要求派生类必须实现 `Area()`，这里通常填 `= 0`。

### 2. 关键字 `const`

来自 2022A 卷运算符重载题。

```cpp
class CInstan {
public:
    void Display() const;
};

void CInstan::Display() _______
{
    cout << "t1 = " << t1 << ", t2 = " << t2 << endl;
}
```

应填：

```cpp
const
```

判断依据：类内声明是 `void Display() const;`，类外定义也必须保持一致。

### 3. 关键字 `static`

自拟例，贴近填空题常考点。

```cpp
class A {
public:
    ________ int Count();
private:
    static int n;
};
```

应填：

```cpp
static
```

完整形式：

```cpp
static int Count();
```

注意：类外定义静态成员函数时不再写 `static`。

```cpp
int A::Count()
{
    return n;
}
```

### 4. 关键字 `inline`

自拟例。

```cpp
________ int Max(int a, int b)
{
    return a > b ? a : b;
}
```

应填：

```cpp
inline
```

判断依据：短小、频繁调用的函数可能声明为内联函数。类内定义的成员函数通常可视作内联，不一定显式写 `inline`。

### 5. 关键字 `virtual`

自拟例，贴近多态题。

```cpp
class Base {
public:
    ________ void Print();
};

class Derived : public Base {
public:
    void Print();
};
```

应填：

```cpp
virtual
```

判断依据：如果主函数中有 `Base *p = &d; p->Print();` 并要求调用派生类版本，基类函数必须声明为虚函数。

### 6. 函数返回值数据类型

来自 2022A 卷运算符重载题的模式。

```cpp
class CInstan {
public:
    CInstan& operator=(const CInstan&);
};

________ CInstan::operator=(const CInstan& c)
{
    t1 = c.t1;
    t2 = c.t2;
    return *this;
}
```

应填：

```cpp
CInstan&
```

判断依据：赋值运算符常返回当前对象引用，以支持连续赋值，如 `a = b = c;`。

### 7. 作用域限定

来自 2022A 卷 `TS` 类题。

```cpp
class TS {
public:
    TS(int n = 0, int d = 0);
};

_____ TS(int n, int d)
{
    N = n;
    D = d;
}
```

应填：

```cpp
TS::
```

完整形式：

```cpp
TS::TS(int n, int d)
```

判断依据：类外定义成员函数或构造函数时，函数名前要加 `类名::`。

### 8. 函数名，包括运算符重载函数名

自拟例。

```cpp
class Complex {
public:
    Complex operator+(const Complex& c);
};

Complex Complex::________(const Complex& c)
{
    return Complex(real + c.real, imag + c.imag);
}
```

应填：

```cpp
operator+
```

判断依据：返回两个对象相加结果，且声明中已有 `operator+`。

### 9. 函数形参表列的数据类型

来自 2022A 卷后缀自增题。

```cpp
class CInstan {
public:
    CInstan operator++();
    CInstan operator++(int);
};

CInstan CInstan::operator++(________)
{
    int tmp1 = t1, tmp2 = t2;
    t1++;
    t2++;
    return CInstan(tmp1, tmp2);
}
```

应填：

```cpp
int
```

判断依据：后缀 `++` 的形参表中有一个哑元 `int`，前缀 `++` 无参数。

### 10. 构造函数初始化列表

来自 2022A 卷 `Circle : public Point` 题。

```cpp
class Circle : public Point {
public:
    Circle(int x, int y, int r);
private:
    double R;
};

Circle::Circle(int x, int y, int r) _________
{
    R = r;
}
```

应填：

```cpp
: Point(x, y)
```

完整形式：

```cpp
Circle::Circle(int x, int y, int r) : Point(x, y)
{
    R = r;
}
```

判断依据：派生类构造函数要先调用基类构造函数初始化基类部分。

## 0.3.2 类型名后挖出一行的一部分

### 变量名或变量初始化/构造

自拟例。

```cpp
int ________;
for (i = 0; i < 10; i++)
    sum += a[i];
```

应填：

```cpp
i = 0, sum = 0
```

完整形式：

```cpp
int i = 0, sum = 0;
```

另一个类对象例：

```cpp
Point ________;
p.Print();
```

应填：

```cpp
p(1, 2)
```

完整形式：

```cpp
Point p(1, 2);
```

判断依据：看后面用了哪些变量、对象，以及是否需要初值。

## 0.3.3 变量/函数名前挖出一行的一部分

### 1. 类名作用域限定

自拟例，静态成员常这样考。

```cpp
class A {
public:
    static int n;
};

int main()
{
    ________ n = 10;
}
```

应填：

```cpp
A::
```

完整形式：

```cpp
A::n = 10;
```

判断依据：`n` 是类的静态成员，类外访问时可以用 `类名::成员名`。

### 2. 命名空间作用域限定

自拟例。

```cpp
namespace N {
    int x = 5;
}

int main()
{
    cout << ________ x << endl;
}
```

应填：

```cpp
N::
```

完整形式：

```cpp
cout << N::x << endl;
```

判断依据：变量在命名空间 `N` 中定义，未写 `using namespace N;` 时要用 `N::`。

### 3. 类型名

自拟例。

```cpp
________ p = new int[10];
delete [] p;
```

应填：

```cpp
int*
```

完整形式：

```cpp
int* p = new int[10];
```

判断依据：后面用 `delete []` 释放，说明前面声明的是动态数组指针。

## 0.3.4 声明时类名后挖出一行的一部分

### 继承关系

来自 2022A 卷 `Circle` 题模式。

```cpp
class Point {
public:
    void Print();
};

class Circle ________ {
public:
    void Print();
};
```

应填：

```cpp
: public Point
```

完整形式：

```cpp
class Circle : public Point
```

判断依据：后续若在 `Circle` 中调用 `Point::Print()`，或构造函数初始化列表写 `Point(x, y)`，说明 `Circle` 是 `Point` 的派生类。

## 0.3.5 `return` 后挖出一行的一部分

### 1. 返回形参中的流对象

自拟例，输入输出运算符重载常考。

```cpp
ostream& operator<<(ostream& out, const Complex& c)
{
    out << c.real << "+" << c.imag << "i";
    return ________;
}
```

应填：

```cpp
out
```

判断依据：`operator<<` 返回 `ostream&`，要返回传入的输出流，以支持连续输出。

### 2. 返回具名对象或变量

自拟例。

```cpp
int Max(int a, int b)
{
    int m = a > b ? a : b;
    return ________;
}
```

应填：

```cpp
m
```

判断依据：函数返回值类型为 `int`，前面已经算出具名变量 `m`。

### 3. 返回匿名对象

来自 2022A 卷 `TS Div` 题模式。

```cpp
TS Div(TS& r1, TS& r2)
{
    return __________________________;
}
```

可填：

```cpp
TS(r1.N / r2.D + r1.D / r2.N, r1.D / r2.D)
```

判断依据：函数返回值类型为 `TS`，没有现成对象时可以临时构造一个匿名对象返回。

### 4. 返回 `this` 或 `*this`

来自 2022A 卷赋值运算符题。

```cpp
CInstan& CInstan::operator=(const CInstan& c)
{
    if (this == &c)
        return *this;
    t1 = c.t1;
    t2 = c.t2;
    return ________;
}
```

应填：

```cpp
*this
```

判断依据：返回类型是 `CInstan&`，所以返回当前对象本身，即 `*this`；如果返回类型是 `CInstan*`，才返回 `this`。

## 0.3.6 分支/循环条件小括号中挖出一行的一部分

### 1. 文件成功打开的检查

自拟例。

```cpp
ifstream fin("data.txt");
if (________) {
    cout << "open error" << endl;
    return 0;
}
```

应填：

```cpp
!fin
```

或：

```cpp
fin.fail()
```

判断依据：刚打开文件，还没读取时，通常先检查打开是否失败。

### 2. 文件结尾检查

自拟例。

```cpp
int x;
while (________) {
    cout << x << endl;
}
```

较稳妥填法：

```cpp
fin >> x
```

完整形式：

```cpp
while (fin >> x) {
    cout << x << endl;
}
```

如果题目已经在循环体内读取，也可能填：

```cpp
!fin.eof()
```

但优先用 `while (fin >> x)`，它能避免最后一次读取失败后重复处理旧值。

### 3. 一般条件

自拟例。

```cpp
if (________)
    cout << "positive" << endl;
else
    cout << "non-positive" << endl;
```

应填：

```cpp
x > 0
```

判断依据：根据分支输出、循环体语义反推条件。

## 0.3.7 类/命名空间声明中挖出一整行

### 1. 友元类/函数声明

来自 2022A 卷 `TS Div` 题。

```cpp
class TS {
public:
    TS(int n = 0, int d = 0);
    void Print();
    _________________________
private:
    int N, D;
};

TS Div(TS& r1, TS& r2)
{
    return TS(r1.N / r2.D + r1.D / r2.N, r1.D / r2.D);
}
```

应填：

```cpp
friend TS Div(TS&, TS&);
```

判断依据：全局函数 `Div` 访问了 `TS` 的私有成员 `N, D`，所以要声明为友元函数。

### 2. 成员函数声明

自拟例。

```cpp
class A {
public:
    _________________________
private:
    int x;
};

void A::Set(int n)
{
    x = n;
}
```

应填：

```cpp
void Set(int n);
```

判断依据：类外出现了 `void A::Set(int n)` 的成员函数定义，类内必须先声明。

### 3. 成员变量声明

自拟例。

```cpp
class Point {
public:
    Point(double x, double y) { X = x; Y = y; }
private:
    _________________________
};
```

应填：

```cpp
double X, Y;
```

判断依据：构造函数体中使用了 `X`、`Y`，它们应是类的数据成员。

### 4. 虚函数声明

自拟例，贴近多态题。

```cpp
class Animal {
public:
    _________________________
};

class Dog : public Animal {
public:
    void Speak() { cout << "Dog" << endl; }
};

int main()
{
    Dog d;
    Animal *p = &d;
    p->Speak();
}
```

应填：

```cpp
virtual void Speak();
```

若基类不实现，则可填：

```cpp
virtual void Speak() = 0;
```

判断依据：基类指针调用同名函数并希望动态绑定到派生类版本。

### 5. 权限划分

自拟例。

```cpp
class A {
    int x;
    ________
    void Set(int n) { x = n; }
};
```

应填：

```cpp
public:
```

判断依据：类默认权限是 `private`，若主函数中要调用 `a.Set(3);`，`Set` 必须放在 `public:` 区域。

## 0.3.8 函数体末尾挖出一整行

### 1. `return` 语句

来自 2022A 卷前缀自增题。

```cpp
CInstan CInstan::operator++()
{
    cout << "++test" << endl;
    t1++;
    t2++;
    _________________________;
}
```

可填：

```cpp
return *this;
```

或：

```cpp
return CInstan(t1, t2);
```

判断依据：前缀 `++` 先改变自身，再返回改变后的值。

### 2. 私有成员赋值或开辟空间

来自 2022A 卷 `Circle::Set` 题。

```cpp
void Circle::Set(double x, double y, double r)
{
    Point::Set(x, y);
    ________________;
}
```

应填：

```cpp
R = r
```

完整形式：

```cpp
R = r;
```

判断依据：函数参数中有半径 `r`，基类部分已经由 `Point::Set` 设置，剩下派生类成员 `R` 还没赋值。

开辟空间例：

```cpp
String::String(const char* s)
{
    len = strlen(s);
    _________________________
    strcpy(str, s);
}
```

应填：

```cpp
str = new char[len + 1];
```

### 3. 成员数组空间释放

自拟例。

```cpp
String::~String()
{
    _________________________
}
```

应填：

```cpp
delete [] str;
```

判断依据：构造函数中若 `str = new char[...]`，析构函数中要用 `delete []` 释放。

### 4. 全局函数的计算

自拟例。

```cpp
void Swap(int& a, int& b)
{
    int t = a;
    a = b;
    _________________________
}
```

应填：

```cpp
b = t;
```

判断依据：无返回值函数，按函数功能补完最后一步计算。

## 0.3.9 全局部分挖出一整行

### 1. 静态变量初始化

自拟例。

```cpp
class A {
public:
    static int count;
};

_________________________

int main()
{
    cout << A::count << endl;
}
```

应填：

```cpp
int A::count = 0;
```

判断依据：静态数据成员必须在类外定义并初始化，普通 `static const int` 特例除外。

### 2. 头文件包含

自拟例。

```cpp
_________________________
using namespace std;

int main()
{
    cout << "hello" << endl;
}
```

应填：

```cpp
#include <iostream>
```

文件流例：

```cpp
_________________________
using namespace std;

int main()
{
    ifstream fin("a.txt");
}
```

应填：

```cpp
#include <fstream>
```

### 3. `using namespace`

自拟例。

```cpp
#include <iostream>
_________________________

int main()
{
    cout << "OK" << endl;
}
```

应填：

```cpp
using namespace std;
```

判断依据：代码中直接写 `cout`、`endl`，没有写 `std::cout`、`std::endl`。

### 4. 流对象的构建

自拟例。

```cpp
#include <fstream>
using namespace std;

int main()
{
    _________________________
    fout << "hello" << endl;
    fout.close();
}
```

应填：

```cpp
ofstream fout("out.txt");
```

输入文件例：

```cpp
ifstream fin("in.txt");
```

判断依据：后续对 `fout` 或 `fin` 进行文件流操作，但此前没有定义流对象。

### 5. 类/全局对象声明

自拟例。

```cpp
_________________________

void f(A& a);

class A {
};
```

应填：

```cpp
class A;
```

判断依据：类 `A` 在完整定义前已经作为引用或指针类型使用，需要提前声明。

## 0.3.10 主函数挖出一整行

### 1. 打开文件

自拟例。

```cpp
int main()
{
    ifstream fin;
    _________________________
    int x;
    fin >> x;
}
```

应填：

```cpp
fin.open("data.txt");
```

也可直接在定义时打开：

```cpp
ifstream fin("data.txt");
```

### 2. 变量初始化

自拟例。

```cpp
int main()
{
    _________________________
    cin >> n;
    for (int i = 1; i <= n; i++)
        sum += i;
    cout << sum << endl;
}
```

应填：

```cpp
int n, sum = 0;
```

判断依据：后续使用 `n` 和 `sum`，且 `sum` 做累加前必须初始化。

### 3. 格式化输出

自拟例。

```cpp
int main()
{
    double x = 3.14159;
    _________________________
}
```

应填：

```cpp
cout << fixed << setprecision(2) << x << endl;
```

若使用 `setw`、`setfill`，记得包含：

```cpp
#include <iomanip>
```

### 4. 关闭文件

自拟例。

```cpp
int main()
{
    ofstream fout("out.txt");
    fout << "hello" << endl;
    _________________________
}
```

应填：

```cpp
fout.close();
```

判断依据：前面打开了文件流，主函数结尾常要求关闭。

### 5. `delete` 语句

自拟例。

```cpp
int main()
{
    int *p = new int[10];
    _________________________
}
```

应填：

```cpp
delete [] p;
```

如果是单个对象：

```cpp
Point *p = new Point(1, 2);
delete p;
```

判断依据：`new` 对应 `delete`，`new []` 对应 `delete []`。
